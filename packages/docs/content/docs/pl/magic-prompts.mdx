# Prompty systemowe

OpenChamber używa w tle wbudowanych promptów za każdym razem, gdy robi coś automatycznie — pisze komunikat commita, szkicuje PR, przegląda issue, rozwiązuje konflikt, podsumowuje sesję. Prompty systemowe to miejsce, w którym czytasz i przepisujesz te prompty. Otwórz je w **Settings → Prompty systemowe**.

Do normalnego użytku ta strona nie jest potrzebna. Sięgnij po nią, gdy chcesz, by przepływ zachowywał się inaczej — powiedzmy komunikaty commitów w określonym stylu.

## Edytuj prompt

1. Otwórz **Settings → Prompty systemowe**.
2. Wybierz prompt z grup w pasku bocznym — Git, GitHub, Planning i Session.
3. Edytuj tekst i zapisz.

Niektóre prompty mają część widoczną (wiadomość, którą zobaczysz) i część z instrukcjami (ukryte wskazówki dla agenta). Prompty mogą zawierać `{{placeholders}}`, które OpenChamber uzupełnia, jak diff czy tytuł issue — zostaw je na miejscu.

## Reset

Zmieniłeś zdanie? Każdy prompt ma **reset to default**, a jest też **reset all**, jeśli chcesz zacząć wszystko od nowa.
